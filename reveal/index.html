第8組報告

主題：什麼是 Git、GitHub、Gist 與 cmsimde？

成員：6、7、12、13、19、42

什麼是 Git？

Git 是一種 版本控制系統。它可以追蹤程式碼的變更歷史，能回到過去版本，也讓多人同時協作而不混亂。

為什麼要用 Git？

  • 版本控制，可回復歷史版本。
  • 支援多人協作，不會覆蓋他人工作。
  • 能建立分支，方便開發新功能。

怎麼使用 Git？

  1. git init 初始化儲存庫
  2. git add . 添加變更
  3. git commit -m "更新" 提交變更
  4. git push origin main 推送至遠端

什麼是 GitHub？

GitHub 是一個基於 Git 的 雲端代碼託管平台，提供遠端儲存、協作開發、Issue 追蹤、Pull Request 等功能。

為什麼要用 GitHub？

  • 雲端儲存程式碼，隨時可訪問。
  • 多人協作與代碼審查。
  • 支援開源專案的貢獻與管理。

怎麼使用 GitHub？

  1. 註冊 GitHub 帳號。
  2. 建立儲存庫。
  3. 將本地 Git 專案推送至 GitHub。
  4. 使用 git push origin main 上傳代碼。

什麼是 Gist？

Gist 是 GitHub 提供的服務，用來分享與管理小段程式碼或文件。

為什麼要用 Gist？

  • 方便快速分享小段程式碼。
  • 可作為個人代碼片段庫。
  • 支援公開或私有選擇。

怎麼使用 Gist？

  1. 登入 GitHub 帳戶並訪問 Gist。
  2. 建立新的 Gist 並儲存代碼。

什麼是 cmsimde？

cmsimde 是一個結合 GitHub Pages 的 簡易內容管理系統（CMS）。 使用 Markdown + GitHub 當後台，省去傳統資料庫與伺服器設定。
